# 🍃 Modern Spring Boot Backend Study

Java 21과 Spring Boot를 활용하여 견고한 백엔드 애플리케이션을 구축하고, 자동화된 인프라 환경을 경험하는 스터디 조직입니다. 단순히 기능 구현에 그치지 않고, **운영 가능한 서비스**를 만드는 것을 목표로 합니다.

## 🎯 Core Objectives

- **Modern Java**: Java 21의 최신 기능을 활용한 효율적인 코드 작성
- **Clean Architecture**: 유지보수가 용이한 계층화 구조 설계 및 구현
- **Robust Backend**: Spring Boot 기반의 안정적인 API 및 비즈니스 로직 개발
- **Full-cycle DevOps**: GitHub Actions를 이용한 CI/CD 파이프라인 구축 및 무중단 배포 경험

## 🔄 Issue-Based Workflow

우리는 리더와 멤버 간의 명확한 커뮤니케이션을 위해 다음 프로세스를 엄격히 준수합니다.

1.  **Issue Creation**: 리더가 각 멤버의 리포지토리에 작업 요구사항을 **Issue**로 할당합니다.
2.  **Branching**: 멤버는 할당된 이슈 번호를 포함하여 브랜치를 생성합니다.
    - `feature/{Name}/{Issue-Number}` (예: `feature/haechan/12`)
    - **주의**: 모든 브랜치 명은 반드시 \*\*영어(English)\*\*로 작성합니다.
3.  **Implementation & PR**: 기능 구현 후 `main` 브랜치로 **Pull Request**를 생성합니다.
4.  **Leader Review**: 리더가 PR의 코드를 확인하고 피드백을 남깁니다.
5.  **Merge**: 리뷰가 승인되면 `main`에 병합(Merge)됩니다.
